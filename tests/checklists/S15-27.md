## Step S15~S27 테스트 체크리스트

1. **검색/활동 피드 (S15)**  
   - [ ] 다양한 필터(폴더, 상태, 검색어)로 `/api/search`가 적절한 문서를 반환하는지 검증.  
   - [ ] 권한 없는 사용자/비로그인 요청이 403/401인지 확인.  
   - [ ] `/api/workspaces/{id}/activity-feed`가 시간순 정렬·pagination을 유지하는지 검증.

2. **Export 흐름 (S16~S17)**  
   - [ ] Export 요청 생성과 queue 상태 트래킹.  
   - [ ] `jobId` 조회 결과가 이후 상태 변경(완료/실패) 반영.  
   - [ ] 너비/권한/invalid ID 인풋은 400/403 처리.  
   - [ ] Cancel/Retry 엔드포인트가 idempotent하게 작동.

3. **콜라보레이션/멘션 (S18~S21)**  
   - [ ] Presence 상태 채널(예: WebSocket)을 열고 cursor/selection 공유.  
   - [ ] Mention/Activity API는 적절한 role 기반으로 결과 제한.  
   - [ ] Media upload(Embed) API는 signed URL, multipart, size 제한을 준수.

4. **Realtime/Offline (S22~S23)**  
   - [ ] Realtime heartbeat/presence reconnection 테스트.  
   - [ ] Offline sync 시 stalemate 상황에서 diff/merge 로직 유효성.  
   - [ ] IndexedDB/memory-level 캐시에서 queue 동작 확인.

5. **워크플로우·트랙체인지·공유( S24~S26)**  
   - [ ] Workflow status transition에서 reviewer 알림/이력 생성.  
   - [ ] Track Changes accept/reject 이벤트와 inline diff.  
   - [ ] 고급 공유 정책(도메인 제한/role matrix) 적용 후 문서 접근 검증.

6. **AI/Slash (S27)**  
   - [ ] Slash palette로 명령어 호출/추천; rate limit(1분당 요청 제한) 테스트.  
   - [ ] AI 응답 stream/rollback 경로 테스트.  
   - [ ] Audit trail 엔드포인트가 AI 이벤트(log/audit) 기록.

각 Step별로 위 체크리스트를 참고해서 Vitest/Playwright(필요 시) 스위트를 작성하고, 완료된 단계는 본 체크리스트에서 체크 표시 후 `docs/planning/development-plan.md`에 반영합니다.
