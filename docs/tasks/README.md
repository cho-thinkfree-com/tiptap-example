# 작업 인덱스

이 디렉터리는 마일스톤 단위로 관리하기에는 세부적인 작업 항목을 추적하기 위한 공간입니다. 각 파일은 하나의 기능, 리팩터링, 조사 주제에 집중하세요.

## 네이밍 규칙
- `YYYY-MM-DD-short-slug.md` 형식을 사용합니다 (예: `2025-11-05-block-dragging.md`).
- slug는 하이픈을 사용한 소문자 조합으로 간결하게 작성하세요.

## 파일 템플릿
```
# Task Title
- **Owner:** 이름 또는 TBD
- **Status:** Planned / In Progress / Blocked / Done
- **Related Docs:** 관련 스펙, 계획, 이슈 링크

## Context
간단한 배경 설명.

## Checklist
- [ ] Step 1
- [ ] Step 2

## Notes
결정 사항, blocker, 후속 조치.
```

새 작업 파일을 만들 때마다 이 README에 활성 작업 표를 추가해 추적하세요.
