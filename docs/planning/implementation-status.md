# 구현 상태 업데이트

## 백엔드
- **API 현황**: 인증, 워크스페이스, 멤버, 문서, 폴더, 감사 로그 등 핵심 기능에 대한 API 엔드포인트가 `server/src/app.ts`에 구현되어 있습니다. 각 기능은 `server/src/modules` 아래의 서비스와 레포지토리로 분리되어 관리됩니다.
- **API 문서**: `docs/api/openapi`에 OpenAPI 명세가 단계별로 저장되어 있으며, `step-S14.yaml`이 현재 감사 로그 API까지의 명세를 반영합니다.

## 프론트엔드 (UI 개편 후)
- **새로운 아키텍처**: `pages`와 `layouts` 디렉토리를 중심으로 UI 구조를 재설계했습니다. `react-router-dom`을 사용하여 페이지 간 라우팅을 관리하며, 인증된 사용자를 위한 보호된 라우트(`ProtectedRoute`)를 구현했습니다.
- **디자인 시스템**: `src/styles/theme.ts`에 정의된 커스텀 MUI 테마를 전역으로 적용하여 일관된 디자인을 유지합니다.
- **인증 흐름**: 사용자는 새롭게 디자인된 로그인 (`src/pages/auth/LoginPage.tsx`) 및 회원가입 (`src/pages/auth/SignupPage.tsx`) 페이지를 통해 인증을 처리할 수 있습니다. `AuthContext`가 인증 상태, 로딩, 에러 처리를 담당합니다.
- **워크스페이스 대시보드**: 로그인 후 진입하는 대시보드(`src/pages/dashboard/WorkspaceDashboardPage.tsx`)는 사용자의 워크스페이스 목록을 보여주며, 워크스페이스가 없을 경우 생성을 유도합니다. 워크스페이스 생성 기능은 다이얼로그(`CreateWorkspaceDialog.tsx`)를 통해 제공됩니다.
- **메인 레이아웃**: `src/layouts/MainLayout.tsx`는 개별 워크스페이스 뷰의 핵심 레이아웃입니다. 반응형 사이드바를 포함하며, 사이드바에는 다른 워크스페이스로 전환할 수 있는 드롭다운 메뉴와 프로필 정보 및 로그아웃 기능이 포함된 메뉴가 구현되어 있습니다.
- **개별 워크스페이스 뷰**: `src/pages/workspace/WorkspacePage.tsx`)는 개별 워크스페이스의 폴더와 문서 목록을 보여주는 "드라이브" 형태의 UI를 제공합니다. 폴더 탐색, 문서 및 폴더 이름 변경/삭제 기능이 구현되었습니다.
- **에디터 페이지**: `src/pages/editor/EditorPage.tsx`는 문서의 제목과 내용을 불러와 Tiptap 에디터로 렌더링합니다. 에디터의 레이아웃은 `EditorLayout.tsx`에서 관리하며, 제목 수정이 가능합니다. 자동 저장 기능과 "닫기" 버튼이 구현되었습니다.

## 다음에 개발해야 할 내용

### 1. 핵심 기능 구현
- **드라이브 탐색 기능**: `WorkspacePage`에서 폴더를 클릭했을 때 해당 폴더의 내용이 표시되도록 하고, 경로에 따라 브레드크럼이 업데이트되어야 합니다. (완료)
- **에디터 저장 기능**: 에디터 페이지에서 문서 제목과 내용을 변경한 후, "저장" 버튼 또는 자동 저장 기능으로 서버에 변경 사항을 반영해야 합니다 (`updateDocument`, `appendRevision` API 연동). (완료)
- **최근 문서 목록**: 대시보드와 워크스페이스 페이지의 "최근 문서" 목록을 실제 데이터와 연동해야 합니다. 이를 위해 새로운 API 엔드포인트 개발이 필요할 수 있습니다. (완료)

### 2. 추가 기능 개발
- **워크스페이스 설정**: 사이드바의 "Settings" 버튼을 클릭했을 때, 워크스페이스 이름과 설명을 수정할 수 있는 기능을 구현해야 합니다 (`updateWorkspace` API 연동). (완료)
- **멤버 관리**: "Members" 버튼 클릭 시, 워크스페이스 멤버를 초대, 삭제하고 역할을 변경할 수 있는 관리 페이지를 구현해야 합니다 (멤버 관련 API 연동). (완료)

### 3. UI/UX 개선
- **예상치 못한 로그아웃 현상**: 브레드크럼 링크 클릭 시 발생하는 로그아웃 현상을 조사하고 해결해야 합니다. (해결됨)
- **동적 페이지 제목**: `AppBar`의 제목이 현재 페이지나 컨텍스트에 맞게 동적으로 변경되도록 개선합니다. (완료)
- **레이아웃 반응형 동작 개선**: 작은 화면에서 사이드바가 열렸을 때 콘텐츠 영역이 적절히 반응하도록 레이아웃을 수정합니다. (완료)
- **로딩 상태 개선**: 데이터 로딩 시, 스켈레톤 UI 등을 적용하여 사용자 경험을 향상시킵니다. (완료)
- **파일/폴더 관리 UI 개선**: 파일 및 폴더 목록을 테이블 형태로 표시하고, 이름 변경/삭제 메뉴의 UI/UX를 개선했습니다. (완료)
- **문서 마지막 수정 정보**: 문서 목록에 마지막 수정한 사람 정보와 상세한 수정 시간을 표시합니다. (완료)
